Example callback for allocate function local context.

It just does followings.

  • check func->impl == 1
  • register cb_exec as function executor.
  • register cb_free as function local context de-allocator.

See also rt_initialize_context

Variable Documentation

unsigned char affine_nnb[]
static

NNB format binary data from Affine_000.nntxt.

By default just call rt_add_callback to use call back function.

But here is some modification to use callback more flexible.

To use callback, you must set 'function implement flag' in NNB file. You can get template setting file with following command.

1 $ cp build/test/nnabla/succeed/Affine_000.nntxt .
2 $ nnabla_cli nnb_template Affine_000.nntxt Affine_000.yaml

Then you should be got following 'Affine_000.yaml'.

1 functions:
2  Affine:
3  implement: 0
4 variables:
5  x: FLOAT32
6  weight: FLOAT32
7  bias: FLOAT32
8  y0: FLOAT32

Edit 'Affine_000.yaml' and save as 'settings.yaml'

1 functions:
2  Affine:
3  implement: 1

NOTE: If you DON'T want to use callback, set implements to 100.

And convert nntxt to nnb again with settings.yaml.

1 $ nnabla_cli convert -s settings.yaml
2 nnabla_cli convert -s settings.yaml Affine_000.nntxt Affine_000.nnb

Then we create binary data with following command.

1 $ xxd -i <Affine_000.nnb >Affine_000.c
